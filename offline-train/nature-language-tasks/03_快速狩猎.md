# 快速狩猎

## 任务来源

- 代码任务：`src.tasks.QuickHuntTask.QuickHuntTask`
- 任务名称：快速狩猎
- 任务描述：从首页进入快速狩猎，调度米饭并补充数量最少的属性圣石。
- 任务分组：`日常/周常`

## 总体顺序

1. 确认游戏首页。
2. 检查 1920x1080 参考点 `(1782, 237)` 是否为红色。
3. 非红色时按今日已完成跳过。
4. 红色时点击该相对点并 OCR `狩猎场`，确认进入快速狩猎。
5. 在地图区域内最多向下滚轮 `6` 次，通过 OCR 定位目标狩猎场。
6. 按配置调度狩猎场和金币/经验（史莱姆）航线米饭。
7. 读取火、水、风、光、暗五种圣石数量，选择数量最少的属性洞穴。
8. 米饭流程失败时仍继续执行圣石流程。
9. 使用固定返回点返回，并联合确认首页小屋图标、小屋亮度和抽抽乐 OCR。

## 米饭流程

1. 米饭为零时跳过狩猎场和冒险航线。
2. 狩猎场选择不再使用章节专用小区域或拖动：鼠标位于地图大区域内，逐次向下滚轮并 OCR `野猪洞穴`、`蜥蜴人祭坛`、`守山人休息处`，最多滚轮 `6` 次，点击配置目标的 OCR 框中心。
3. 不再点击或 OCR `哥布林遗迹`，冒险航线和圣石流程均不执行地图复位。
4. `狩猎场x1 / 双倍图MAX`：狩猎场使用 `MIN`，选中的双倍冒险航线使用 `MAX`。
5. `狩猎场MAX / 跳过冒险航线`：狩猎场使用 `MAX`，金币和经验航线全部跳过。
6. 不再执行剩余米饭狩猎场兜底。

## 双倍判断

1. 只使用根目录模板 `Double.png`，不再使用金币/经验双倍和非双倍四张旧模板。
2. 在双倍区域保留所有有效匹配；区域上半部分命中代表金币双倍，下半部分命中代表经验/史莱姆双倍，上下同时命中代表两个都双倍。
3. `忽视双倍`：不匹配模板，固定选择金币。
4. `优先双倍`：哪个双倍就刷哪个；两个都双倍时优先金币；两个都不双倍时跳过冒险航线。
5. `强制双倍`：优先配置的资源；首选不双倍时使用另一个双倍资源；两个都不双倍时跳过。

## 圣石流程

1. 点击 `(177, 449)` 进入属性洞穴，不执行地图复位。
2. 火把数量沿用米饭数量 OCR 区域。
3. 在五属性数量区域从上到下读取恰好五个数字，依次映射为火、水、风、光、暗。
4. 选择数值最小的属性；数量并列时按火、水、风、光、暗的从上到下顺序选择第一个。
5. 在左侧属性列表 OCR 并点击对应的“某之洞穴”框中心。
6. 执行公共流程：`快速狩猎 → MAX → 狩猎 → 等待结算`。

## 测试 UI

快速狩猎配置展开后提供四组测试按钮。按钮会把对应动作作为一次性任务加入
任务执行器队列，不会在界面线程中直接执行自动化。

1. `快速狩猎入口测试`
   - `只读检查入口`：截取当前画面，同时确认首页小屋按钮、首页亮度、抽抽乐 OCR，以及 1920x1080 参考点 `(1782, 237)` 的像素是否为红色；不移动或点击鼠标。
   - `打开狩猎菜单`：确认首页后等待 `0.5` 秒；检查参考点 `(1782, 237)`，非红色时按今日已完成跳过；红色时再等待 `0.5` 秒，点击该相对点并等待 `1` 秒，随后持续 OCR 原版识别词 `狩猎场` 以确认进入快速狩猎。
   - 使用前应停留在游戏首页。
2. `快速狩猎菜单测试`
   - `只读检查菜单`：在同一帧中检查菜单 OCR、地图大区域 OCR、五属性圣石数量、收起按钮模板和通用 `Double.png` 多目标匹配；不移动或点击鼠标。
   - `执行米饭(消耗)`：按当前章节、资源倾向、双倍策略和米饭分配配置执行米饭流程，会实际消耗米饭。
   - 使用前应已经打开快速狩猎菜单。
3. `快速狩猎圣石测试`
   - `执行圣石(消耗)`：读取五属性数量并选择最少的属性，以 MAX 执行，会实际消耗火把。
   - `返回主页`：只执行快速狩猎的有限次数返回主页流程。
   - 使用前应已经打开快速狩猎菜单或停留在需要返回的快速狩猎子页面。
4. `快速狩猎完整测试`
   - `完整执行(消耗)`：从游戏首页执行入口、米饭、圣石和返回主页完整流程，会实际消耗米饭和火把。

任务运行期间再次点击测试按钮不会插入第二个动作，而是提示先停止当前任务。每次
测试的阶段、OCR 文本、模板分数、像素相似度和最终结果会显示在任务状态区域。
