# PyAppify release flow

`ok-bd2` follows the release shape used by `BnanZ0/ok-nte`.

## What the installed EXE does

The user downloads a single installer such as `ok-bd2-win32-China-setup.exe`.
After installation, Windows starts an outer launcher `ok-bd2.exe`. That launcher
is generated by PyAppify. It opens the white update window, checks the configured
update repository, installs Python dependencies when needed, syncs the app code
into its working directory, and then runs `main.py`.

The dark `ok-script` window is still created by the existing application entry:

```text
ok-bd2.exe -> PyAppify launcher/update window -> main.py -> ok.OK(config).start()
```

No separate update UI is needed in `main.py`.

## Source repository vs update repository

The main repository keeps source code, tests, docs, and training material. The
update repository is intentionally smaller and is generated during release.

`deploy.txt` is the sync list for that update repository. During the release
workflow, `python -m ok.update.inline_ok_requirements --tag <tag>` copies
`ok-script` and `pyappify` into local `ok/` and `pyappify/` directories, removes
the external `ok-script` dependency from the update requirements, and updates
`src/config.py` from `version = "dev"` to the tag being released.

## Required repositories and secrets

Create these repositories before publishing a tag:

```text
https://github.com/GodRaymond233/ok-bd2
https://github.com/GodRaymond233/ok-bd2-update
https://cnb.cool/GodRaymond233/ok-bd2-update.git
```

Configure these GitHub Actions secrets:

```text
OK_GH     GitHub token that can push to GodRaymond233/ok-bd2-update
CNB_GH    CNB token that can push to cnb.cool/GodRaymond233/ok-bd2-update.git
```

Signing is disabled by default. If `SIGN_BUILD` or `SIGN_SETUP` is changed to
`true`, also configure:

```text
SIGNPATH_API_TOKEN
SIGNPATH_ORGANIZATION_ID
SIGNPATH_SIGNING_POLICY_SLUG
```

## Publishing

Use a stable semantic version tag. The build workflow only creates installer
assets for tags like `v0.1.0`.

```powershell
git tag v0.1.0
git push origin v0.1.0
```

The workflow will:

1. Install dependencies on Windows.
2. Inline `ok-script` and PyAppify for the update repository.
3. Run the test suite.
4. Sync the update repositories using `deploy.txt`.
5. Build PyAppify installer assets from `pyappify.yml`.
6. Publish the GitHub Release with `pyappify_dist/*`.
